关于Margin的总结

发表于 2019-07-18 更新于 2019-10-23 分类于 CSS 阅读次数：

表现

虽然 margin 可以应用在所有元素，但 display 属性不同时，表现也不同。

  • block 元素可以使用四个方向的 margin 值
  • inline 元素使用上下方向的 margin 值无效
  • inline-block 使用上下方向的 margin 负值看上去无效

注意：inline-block 使用上下方向的 margin 负值只是看上去无效，这与其默认的 vertical-align:baseline 有关系，当垂直对齐的属性值为其他值时，这会显示不同的视觉效果。

margin 为负值的影响

  • 对自身的影响，当元素不存在 width 属性或 width: auto 的时候，负值 margin 会增加元素的宽度。（注意：这仅仅是在元素不存在 width 属性或者 width 属性默认的时候，如果有宽度设置， margin-left 和 margin-right 为负值时，会发生位移。）而 margin-top 为负值的时候，不会增加宽度，而是会让元素上移。 margin-bottom 为负值的时候不会位移，而是会减少自身供 css 读取的高度。
  • 对文档流的影响：元素如果用了 margin-left: -100px; 其会向左位移 100px ，但是和相对定位不同的是，在其后面的元素会进行补位，也就是后面的行内元素会紧贴在此元素之后。如果不脱离文档流不浮动的话， margin 为负值的元素不会破坏页面的文档流。当然，如果你用负 margin 上移一个元素，接下的元素都会跟着上移。
  • 对浮动元素的影响，负 margin 会改变浮动元素的显示位置，圣杯布局、双飞翼布局都是利用这个原理实现的。
  • 对绝对定位的影响。

Margin collapsing

margin collapsing 出现的几种情况：

  • 上面的元素有 margin-bottom，而下面的元素有 margin-top，两个 margin 会重叠，取较大的 margin。
  • 邻近的兄弟姐妹：if you have two elements displaying one after the other in normal flow, the bottom margin of the first element will collapse with the top margin of the following element.
  • 完全空白的 box：If a box is empty, then it’s top and bottom margin may collapse with each other.
  • parent and first or last child element:the margin on the child collapses with any margin on the parent thus ending up on the outside of the parent.
  • 只有普通文档流中块框的垂直外边距才会发生外边距合并。行内框、浮动框或绝对定位之间的外边距不会合并。

避免 margin collapsing 出现的几种方案：

  • Margins never collapse if an item has absolute positioning, or is floated.
  • a box completely empty of content will not collapse it’s top and bottom margin if it has a border,or padding applied.
  • If we add a border to the parent, the margins on the children stay inside.
  • 创建BFC上下文
    • 根元素包含根元素的元素
    • 浮动元素（元素的 float 不是 none）
    • 绝对定位元素（元素的 position 为 absolute 或 fixed ）
    • 行内块元素（元素的 display 为 inline-block）
    • 表格单元格（元素的 display 为 table-cell ， HTML 表格单元格默认为该值）
    • 表格标题（元素的 display 为 table-caption ， HTML 表格标题默认为该值）
    • 匿名表格单元格元素（元素的 display 为 table、table-row、table-row-group、table-header-group、table-footer-group（分别是HTML table、row、tbody、thead、tfoot的默认属性）或 inline-table）
    • overflow 值不为 visible的块元素
    • display 值为 flow-root 的元素
    • contain 值为 layout、content 或 strict 的元素
    • 弹性元素（ display 为 flex 或 inline-flex 元素的直接子元素）
    • 网格元素（ display 为 grid 或 inlien-grid 元素的直接子元素）
    • 多列元素（元素的 column-count 或 column-width 不为 auto ，包括 column-count 为 1 ）
    • column-span 为 all 的元素始终会创建一个新的 BFC ，即使该元素没有抱过在一个多列容器中
  • Flex & Grid 布局

关于 margin 的一些建议

  • 只设置 margin-top 或者 margin-bottom，或只设置一个方向的 margin。

百分比的 margin

当给margin设置百分比的时候，通常会以父元素的内联大小来作为参考。比如，I have a wrapper which is 200 pixels wide, inside is a box which has a 10% margin, the margin is 20 pixels on all sides, that being 10% of 200.

Margins In A Flow-Relative World

  • margin-top = margin-block-start
  • margin-right = margin-inline-end
  • margin-bottom = margin-block-end
  • margin-left = margin-inline-start

We also have two new shorthands which allow for the setting of both blocks at once or both inline.

  • margin-block
  • margin-inline
